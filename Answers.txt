Recommendations to improve signal

1. Make the target type explicit
• Since “100x pct” is consistently selected, codify the inverse mapping to avoid confusion:
   • During training: ensure target = 100 * percent_return (fractional percent).
   • During inference: invert as predRel = yhat / 100.0f.
• Consider persisting this metadata with the model (target type, scale, bias) so it’s not guessed at runtime.

2. Inspect the prediction distribution
• Log histograms or simple summary stats of predsLogRet per batch:
   • min/max/mean/std
   • unique value counts or bucketed counts
• If outputs cluster into a small set of values, look at:
   • Output activation function (e.g., a tanh squashing tightly around a few points).
   • Initialization and learning rate (saturation can occur).
   • Loss scaling and target scaling (if targets are too small/large, gradients can vanish/explode).
   • Regularization and dropout.

3. Calibrate the head
• If you trained on percent returns but standardized/normalized them, ensure you invert that normalization consistently. Right now you’re only testing 4 “raw” inversions, not any z-score inverse (e.g., multiply by std, add mean). If the training used normalization, you should store and apply those stats at inference time. That often improves correlation materially.

4. Evaluate on fixed holdout windows
• Instead of per-batch auto-selection, evaluate on a fixed validation set with a fixed mapping. Use:
   • Correlation
   • Direction accuracy
   • ROC-AUC for direction
   • Calibration curve for magnitude
• This will reduce variance in your diagnostics and make improvements easier to measure.

5. Try a direction-focused head or loss
• If direction is important, add an auxiliary classification head (up/down) with cross-entropy alongside the regression head. Multi-task learning can stabilize directional signal.
• Alternatively, optimize for sign with a differentiable surrogate (e.g., rank loss or hinge-like loss on signed returns).

6. Time alignment and label leakage
• Double-check that the label at t+1 is strictly out-of-sample relative to inputs up to t. Any subtle leakage can inflate MAE while hurting correlation/direction in real deployment.

7. Hyperparameters and architecture
• Increase head capacity slightly or add a small MLP on top of the LSTM output to reduce quantization-like behavior.
• Revisit sequence length, hidden size, dropout, weight decay.
• Consider using a bidirectional LSTM only if you are doing offline evaluation; for online predictions, stick to causal models.
